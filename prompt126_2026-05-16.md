❯ # Juniper Canopy Application Work

## Background

  there is ongoing work adding a juniper-recurrence application that follows the rough outline of the juniper-cascor application but is designed for time-series datasets, particularly irregular dt datasets.
  this model will need to be integrated into the juniper-canopy application which serves as the web front-end, visualization,  and research tool for the Juniper Project.

## Canopy Enhancements

  one of the first, logical steps seems to be to add model selection functionality.
  currently dataset selection exists, but there isn't a clear user facing way to select the desired model.
  a related concern is that there are existing datasets that will incompatible with specific models. e.g., time-series datasets will not be compatible with the juniper cascor model.
  this should be accomplished with greyed-out menu options and explanatory tool-tips or toasts as appropriate.
  the compatibility requirements should also extend in the other direction; if a dataset is selected, selecting an incompatible model should be gated in a similar manner: greyed out menu entries, tool-tips, and toasts.

## Canopy Regressions

  in addition to the new work needed, the canopy application is continuing to demonstrate persistent, fix-resitant regressions that include basic, user-facing, core functionality.
  a significant number of buttons are still not functional.
  a cluster of these issues surrounds the dataset configuration, modification, selection, and application controls in the canopy app

### Significance -- It's Really High

  these regressions have persisted over multiple remediation iterations.
  these regressions are explicitly user-facing and prevent normal user interaction with the app
  these regressions are load-bearing and prevent the proper operation of the application
  these regressions directly prevent the specific research goals and experimental use cases that were the motivation for creating this applicaiton in the first place.
  these regressions are explicitly classified as blockers of the highest level and with the highest priority--none of the previous or ongoing development work is meaningful without a properly functioning canopy application.
