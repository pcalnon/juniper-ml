# juniper-canopy external cascor

## Description

A critical use-case for the juniper-canopy application has not yet been fully realized

### Use Case Requirement

A key requirement for the juniper-canopy application is to provide a temporary monitoring and management front end for an externally running instance of the juniper-cascor application

#### Sub-requirements

1. when juniper-canopy application is launched, in normal mode (not demo mode), it should begin by checking for a running juniper-cascor process
2. if a juniper-cascor process is discovered, the juniper-canopy application should connect to it
3. the juniper-canopy connection should not affect the running process
    - e.g., the juniper-canopy application should not cause the juniper-cacor application parameters, that are able to be modified by juniper-canopy, to be reset to default values
4. the juniper-canopy application should retrieve metrics from the running juniper-cascor application
5. the juniper-canopy application should display the metrics it received from juniper-cascor
6. the juniper-canopy application should retrieve relevant neural network and candidate pool training parameters
7. the juniper-canopy application should display the retrieved parameter values
8. any changes made to juniper-cascor parameters from the juniper-canopy application should be applied to the running instance of the juniper-cascor application
9. all monitoring and visualizations displayed by juniper-canopy should accurately reflect the current, and ongoing, state of the running juniper-cascor application including, but not limited to, the following:
    - network topology
    - network training performance
    - network training state
    - candidate training performance
    - candidate training state
    - dataset visualization
    - decision boundary visualization
10. all training controls should be functional and control the running junipere-cascor application including, but not limited to, the following:
    - stop training
    - start training
    - pause training
    - resume training
    - reset training
11. when the juniper-canaopy application is stopped, closed, shutdown, or terminated, the juniper-cascor application should continue operating normally
    - the juniper-cascor application should contiue running while maintaining the state it was in prior to the juniper-canopy application being stopped
12. the juniper-canopy application should be able to be started and stopped multiple times during a juniper-cascor run
