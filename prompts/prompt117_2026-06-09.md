# Recurrence in Cascor Networks

## Background

the second question, mentioned earlier, concerns the outline of a possible design approach for integrating recurrence with dynamicly constructed cascor networks.
this design is motivated primarily by a desire to process time series data while leaving the dynamic, constructive nature of the cascor training process--particularly the candidate training step--intact, to the extent possible.

## Design Description

in this design, recurrent functionality is added to hidden nodes by introducing a multi-step delay element that borrows the effective functionality of a simple shift register.
the delay element would be added as an additional, final output component of the node.
this approach should leave intact all of the current functionality and structure of cascor hidden nodes--from input weights through the output activation function.
the modified, hidden node would generate its activation output exactly as it currently does.
rather than being passed directly to the downstream nodes, however, node output becomes the input to the node's new recurrence module.
the number of previous samples considered by the module--the bit-wise size of the "shift register" which serves as the time series embedding dimension--is assumed to be finite and bounded.
in practice, the embedding dimension should be on the order of 35.
the recurrent module's output will be an ordered collection of the hidden node's output tensors.
effectively, the new, augmented hidden node--the original hidden node + the recurrence module--should produce an N+1 dimension output tensor that adds a single dimension to the original hidden node's N dimension tensor.

## Integration

using this design, the candidate node training process would proceed normally.
when candidate node pool training has completed and the candidate node(s) with the best correlation selected, the recurrence module will be added to the new hidden node(s) prior to integration into the existing cascor network.

## Specific Questions to Consider

in addition to the feasability and functionality of this recurrent output module approach, specific areas of concern include:

- how correctly, effectively, and accurately would this approach implement recurrence for cascor neural networks?
- how would this approach would affect the network's ability to process irregular Δt datasets?
- how would this model's functional strengths and weaknesses affect or limit the effective horizon of prediction for time series datasets, datasets with irregular Δt, and the existing equities dataset?
- how would this design approach affect mini batch processing?
