## Env var prefix decision

The canonical plan (¬ß3.1, ¬ß3.6) specifies `JUNIPER_WS_*` env var names for kill switches and settings.
Cascor's Settings class uses `env_prefix="JUNIPER_CASCOR_"`, which would produce `JUNIPER_CASCOR_WS_*`.
Since kill switch runbooks and the ops team will reference `JUNIPER_WS_*`, I'll use pydantic-settings `AliasChoices` to accept both, with `JUNIPER_WS_*` as the primary (matching canonical plan).
Existing `ws_max_connections` and `ws_heartbeat_interval_sec` keep their `JUNIPER_CASCOR_*` prefix for backward compat.

### Step 3: Manager sequencing + replay + timeout (0-cascor-2a/2b/3/4/5a/8)

**File:** `src/api/websocket/manager.py` (157 lines) ‚Äî largest change

**`__init__` additions:**

- `server_instance_id: str = str(uuid.uuid4())`
- `server_start_time: float = time.monotonic()`
- `_next_seq: int = 1`, `_seq_lock = threading.Lock()` (threading.Lock, not asyncio ‚Äî used from both sync and async contexts; held O(1) only)
- `_replay_buffer: deque = deque(maxlen=max_replay_buffer_size)`
- `_pending_connections: Set[WebSocket] = set()`
- `_send_timeout_seconds: float`
- New constructor params: `max_replay_buffer_size: int = 1024`, `send_timeout_seconds: float = 0.5`

**New methods:**

- `_assign_seq_and_buffer(message) -> dict`: Under `_seq_lock`, assigns monotonic seq, sets `emitted_at_monotonic`, appends to replay buffer. Returns enriched message copy.
- `replay_since(last_seq) -> list[dict]`: Copy-under-`_seq_lock`. Raises `ReplayOutOfRange` if too old or buffer disabled.
- `connect_pending(websocket) -> bool`: Accept + add to `_pending_connections` (not broadcast-eligible). Sends enriched `connection_established` with `server_instance_id`, `server_start_time`, `replay_buffer_capacity`.
- `promote_to_active(websocket)`: Move from pending to active set.
- Properties: `server_instance_id`, `current_seq`

**Modified methods:**

- `connect()`: Updated `connection_established` payload to include `server_instance_id`, `server_start_time`, `replay_buffer_capacity` (0-cascor-2b)
- `broadcast()`: Calls `_assign_seq_and_buffer(message)` before fan-out (0-cascor-2a)
- `_send_json()`: Wraps `websocket.send_json` in `asyncio.wait_for(timeout=self._send_timeout_seconds)`. Catches `asyncio.TimeoutError` separately for metric (0-cascor-3)
- `broadcast_from_thread()`: Captures future from `run_coroutine_threadsafe`, adds `future.add_done_callback(_log_broadcast_exception)` (GAP-WS-29, 0-cascor-8)
- `disconnect()`: Also removes from `_pending_connections`
- `close_all()`: Also clears `_pending_connections`

**New exception:** `ReplayOutOfRange` (module-level)

### Step 4: Resume protocol in training stream (0-cascor-5b)

**File:** `src/api/websocket/training_stream.py` (62 lines) ‚Äî significant rewrite

New flow:

1. Authenticate
2. `connect_pending()` (not yet broadcast-eligible)
3. Wait up to `ws_resume_handshake_timeout_s` for optional `resume` frame
4. If resume: validate `server_instance_id` match (D-15), call `replay_since()`, send `resume_ok` + replayed events. One resume per connection (D-30).
5. If no resume / timeout / failed: fresh connect (send initial_status + state)
6. `promote_to_active()` ‚Äî now receives broadcasts
7. Keep-alive recv loop (unchanged)

Extract `_handle_resume(websocket, ws_manager, msg) -> bool` helper. Resume failures send `resume_failed` with `reason` field: `server_restarted`, `out_of_range`, `malformed_resume`, `second_resume`.
