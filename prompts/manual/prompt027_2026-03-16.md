# juniper-cascor regressions

3 regressions and 2 issue have been identified in the juniper-canopy application.

## Description

address regressions and issues that have been identified in the juniper-canopy application.

  1. Regression: Metrics Tab, top menu, Dark mode background:
      - on the Training Metrics tab, just below the Training Metrics Heading, the metrics layout top menu does not display a dark background when in dark mode.
      - the image attached to this prompt: Image #11, demonstrates this issue.
  2. Regression: Decision Boundary Tab, stale display:
      - on the Decision Boundary Tab, the decision boundary appears to be calculated prior to beginning of training
      - instead, this plot, and the calculated data behind it, should be recalculated and refreshed each time the Decision Boundary tab is selected
      - additionally, when the decision boundary tab is being displayed, the decision boundary calculations and plot should be refreshed periodically
        - refresh should occur at a reasonable interval
        - if calculations are too time consuming to be refreshed frequently, provide a refresh button so the user can decide when the plot needs to be redrawn
      - the image attached to this prompt: Image #12, demonstrates this issue
  3. Regression: Metrics Tab, Candidate Pool data display
      - a user specified number of previous candidate pool datasets should be displayed
        - (e.g., the most recent 20 candidate pools)
      - each displayed, previous candidate pool datasets should be expandable to allow all of its data to be displayed
      - the links to these previous candidate pool datasets should always be displayed regardless of network training state or display state.
      - the image attached to this prompt: Image #13, demonstrates this issue.
  4. Enhancement: Expandable link in Network information Left menu should be marked as expandable
      - In the Network Information section of the Left Side menu, the Network Information: Details link can be expanded to include more network connection detail
      - however, it isn't immediately obvious that this link can be expanded
      - Modify the display to clearly indicate that this link is expandable using any one or more of the following approaches, or something comparable
        - use drop down arrow (triangle)
        - use the menu header background color change
        - use mouseover with change(s) to font name, type, weight, color,
  5. Enhancement: Metrics Tab, training loss and accuracy plots' display window:
      - the metrics tab displays two live plots of the network training performance
        - training loss over time
        - training accuracy over time
      - data for both plots is displayed using a sliding window of 100 epochs.
        - while this display mode is valid, it should not be the only mode available
      - the following list of display modes should be made available to users:
        - full training history, epoch 0 - current
        - variable epoch display window
          - allow the window width (i.e., number of epochs to display) to be user defined
          - between min-epochs + 1 and max_epochs
        - time between hidden node addition
      - display mode selection should be via radio buttons or comparable.
      - changing display mode during, or after, a training run should regenerate these plots accordingly
