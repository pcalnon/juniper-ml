## correct issues with existing cleanup process

1. the current worktree cleanup procedure is not working properly, as written.
    - the git worktree cleanup process fails when the interactive claude session ends up "trapped" with its CWD set to a non-existent directory path.
    - the interactive claude session should switch to a new, clean git worktree before the old worktree directory is removed.

2. to address this, perform a detailed analysis
    - of the existing worktree cleanup procedure files in the notes/ directory
    - of all worktree cleanup related helper scripts in the scripts/ directory
    - focusing on important process components including:
      - ensuring git worktree branch removal
      - creating another git worktree
      - removing the old worktree directory
      - creating new or fixing and enhancing worktree cleanup helper scripts

3. using the analysis from step 2, develop a detailed plan to create an updated worktree clean up process
    - ensure that the following requirements are met:
      - all changes made in the current claude session are committed and pushed.
        - for the worktree branch
      - then merge the changes into the parent branch
        - the git branch that was active when the git worktree was created.
      - if the parent branch was main:
        - do not merge directly
        - instead open a pull request to merge the worktree branch into main
      - if the parent branch was not main:
        - merge the worktree branch into the parent branch
        - open a pull request to merge the parent branch into main
      - creation of pull requests should include generating a pull request description using the following resources:
        - the pull request template from the notes/templates/ directory
        - previously completed pull request descriptions from the notes/pull_requests/ directory
          - referencing content, layout, and format.
      - the parent branch should again be made the active branch
      - the completed worktree branch should be deleted
      - the cwd should be set to the root of the parent branch
        - not a worktree directory
      - remove the old, completed worktree directory
      - create a new, clean worktree for the current claude session

4. worktree cleanup helper scripts:
    - use the analysis performed in step 2 above
    - evaluate potential updates, modifications, and creations of worktree cleanup scripts
    - add to detailed cleanup procedure update plan
    - include any potentially useful modifications or additions that would benefit any aspect of the worktree cleanup process

5. write the plan into a markdown file in the notes/ directory

6. use sub-agents to validate all aspects of the new cleanup process and its development plan
    - verify procedure steps against best practices and documentation
    - add tests as necessary to validate accuracy and correctness of the process and related scripts

7. once the plan has been completed, validated, and written to a file, implement the plan
    - update the plan with implementation progress and identified issues or concerns.

8. perform a final round of testing and validation of the new cleanup procedure and associated scripts.
    - use "clean" sub-agents to perform this work
    - add any new tests needed to complete this evaluation
    - update all references to the previous worktree cleanup procedure file with references to the newly created file.
    - perform final updates to the development plan document
