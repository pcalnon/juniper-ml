# Add worktree directives to CLAUDE.md files

## Primary objective

the primary objective for this prompt is to add worktree usage to the standard agent behavior for all repos in the Juniper Project.

## Generate planning documents

### New Task Procedure

begin by developing a procedure for agents to use when beginning a new task
procedure should incorporate, but not be limited to, the following requirements:

- the agent should create a new working branch
- the agent should open the working branch as a worktree
- the worktree should be created in a unique directory name that incorporates the following
  - branch name
  - repo name
  - date and time of creation
  - any other element required to guarantee the directory name is unique
- the worktree directory should be located in the Juniper directory
- add any additional requirements or components that would improve this procedure

### Task Cleanup Procedure

continue by developing a procedure for agents to use when completing a task
this procedure should assume that worktrees were used to perform the task
procedure should incorporate, but not be limited to, the following elements:

- by default, the working branch, used for the worktree, should be merged into its specific parent branch
  - if best practices and project or application architecture suggests that another branch should be the "merge into" branch:
    - the procedure should incorporate the flexibility to enable this
- after merging is completed, continue cleanup
  - remove the local working branch created during new task worktree setup
  - remove the worktree directory created during new task worktree setup
  - push the "merged into" branch to its repo
- add any additional requirements or components that would improve this procedure

## Deliverables

The following deliverables should be provided for all active repos in the Juniper project.

### Documentation

#### New Task Worktree setup Procedure

the New Task worktree setup procedure should be documented in an appropriately named markdown file in the notes/ directoryu

#### Finished Task Worktree cleanup Procedure

the finished task worktree cleanup procedure should be documented in an appropriately named markdown file in the notes/ directory

### Implementation

#### Update to the AGENT.md file

The New Task Worktree Setup and New Task Worktree Cleanup procedures should be incorporated into the AGENTS.md file

#### Update Claude settings

**New Task Worktree Setup:**

The claude settings should be updated to require the behavior documented in the New Task Worktree Setup procedure.

**Finished Task Worktree Cleanup:**

The claude settings should be updated to require the behavior documented in the Finished Task Worktree Cleanup procedure
