## automated deploy to testpypi and pipy

1. add a new section to CI that manages the package deployment to pypi
the automated deploy should deploy packages to testpypi first.
after waiting period, automated deploy should trigger to deploy packages to pypi.

2. package deployment to pipy should be triggerd for a given repo when:
deploy package when a release is deployed for that repo.

3. this new CI task should be enabled for all repos
adding this to all project CI's will ensure package deployment will be performed for all repos on release
