### Phase 0 Tasks

**Step 0.1: Security Vulnerabilities (4 tasks)**:

0.1.1 — Fix path traversal in snapshot endpoints (CRIT-001)

- File: src/main.py lines ~1144-1169, ~1345
- Add _sanitize_snapshot_name() with regex validation (alphanumeric, hyphens, underscores only)
- Add .resolve() path confinement check ensuring path stays inside _snapshots_dir
- Write targeted security test for ../../, ..\, URL-encoded traversal sequences

0.1.2 — Fix timing attack in API key validation (HIGH-001)

- File: src/security.py line ~57
- Replace `api_key in self._api_keys` (set membership) with hmac.compare_digest()
- Use: `any(hmac.compare_digest(api_key, k) for k in self._api_keys)`

0.1.3 — Suppress internal details in global exception handler (HIGH-002)

- File: src/main.py lines ~274-276
- Log full exception server-side at ERROR level with exc_info=True
- Return generic message to client: "An unexpected error occurred."

0.1.4 — Fix rate limiter memory leak (HIGH-003)

- File: src/security.py lines ~114, ~163-167
- Add _evict_expired() method that removes entries older than window
- Add emergency size cap check (10,000 entries) in check() method
- Write test confirming bounded memory under simulated IP rotation

**Step 0.2: Concurrency Issues (2 tasks)**:

0.2.1 — Fix thread-unsafe CallbackContextAdapter (CRIT-002)

- File: src/frontend/callback_context.py lines ~65-72, ~93-101
- Replace _test_mode and _test_trigger instance attributes with contextvars.ContextVar
- This is preferred over threading.local() since project uses Python >=3.12 and has async code
- Write concurrent callback test verifying thread isolation

0.2.2 — Fix threading.Event replacement race condition (HIGH-004)

- File: src/demo_mode.py line ~1571
- In _perform_reset(), change `self._stop = threading.Event()` to `self._stop.clear()`
- Same for self._pause if applicable
- Write test for rapid stop/reset/start cycling

**Addendum: Backend Concurrency (1 task)**:

0.3.1 — Add threading.Lock to TrainingStateMachine (HIGH-015)

- File: src/backend/training_state_machine.py lines ~84-329
- FSM has no locking despite being accessed from both training thread and REST handlers
- Add self._lock = threading.Lock() and wrap all state mutations in handle_command,
    set_phase, mark_completed, _handle_start, _handle_stop, etc.
