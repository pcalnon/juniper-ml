# Slack Startup problem, Investigate and Fix

## Diagnosis

### Root Cause 1: NVIDIA Userspace Drivers Missing (PRIMARY)

  Your system has a split NVIDIA driver installation - the kernel module is loaded but the userspace libraries are completely absent:

  ┌─────────────────────────────────────────┬─────────┐
  │                Component                │ Status  │
  ├─────────────────────────────────────────┼─────────┤
  │ Kernel module nvidia 580.126.09         │ Loaded  │
  ├─────────────────────────────────────────┼─────────┤
  │ libnvidia-gl-580 (OpenGL/EGL libs)      │ MISSING │
  ├─────────────────────────────────────────┼─────────┤
  │ nvidia-driver-580-open (driver package) │ MISSING │
  ├─────────────────────────────────────────┼─────────┤
  │ nvidia-utils-580 (nvidia-smi, etc.)     │ MISSING │
  └─────────────────────────────────────────┴─────────┘

#### Effect

Electron's GPU process crashes during initialization:
**ERROR:viz_main_impl.cc:189] Exiting GPU process due to errors during initialization

#### Explanation

OpenGL falls back to llvmpipe (CPU software rendering). Vulkan shows only a CPU device.
The software rendering fallback + Wayland results in the Slack window being created but never becoming visible (isVisible: false).

### Root Cause 2: Conflicting slack Package

An unrelated Debian package called slack (v1:0.15.2-11 — "configuration management program for lazy admins") is in a config-files state, leaving behind /etc/slack.conf and /etc/cron.daily/slack.

### Root Cause 3: Dual Slack Installation

Both apt (slack-desktop v4.47.69) and snap (slack v4.46.99) are installed, creating duplicate launcher entries and confusion.

---

## Fix Plan

1. Install nvidia-driver-580-open (pulls in the missing userspace libs)
2. Purge the unrelated slack config management package
3. Remove the snap Slack (keep the newer apt version)
4. Reboot to fully load the NVIDIA userspace
5. Verify Slack launches

### Priority and Analysis

The nvidia driver install and reboot are the critical fix — without GPU userspace libraries, Electron apps can't render windows properly under Wayland.

### Manual Commands

#### Step 1 — Install NVIDIA userspace driver

```bash
sudo apt install -y nvidia-driver-580-open
```

#### Step 2 — Purge the unrelated slack config management package

```bash
sudo dpkg --purge slack
```

#### Step 3 — Remove the snap Slack installation (keep the apt one)

```bash
sudo snap remove slack
```

#### Step 4 — After running all three, reboot when convenient

```bash
sudo reboot
```

#### Step 5 — After reboot, verify with

```bash
# Should show NVIDIA GPU info instead of errors
nvidia-smi
```

#### Step 6 — Should show NVIDIA renderer instead of llvmpipe

```bash
# Should show NVIDIA renderer instead of llvmpipe
glxinfo | grep "OpenGL renderer"
```

#### Step 7 — Launch and show window

```bash
# Should launch and show window
slack
```

### Conclusion

The nvidia driver install is the critical fix. Once libnvidia-gl-580 and the other userspace packages are in place, Electron/Chromium will be able to initialize its GPU process, and Slack's window will render correctly under Wayland.
