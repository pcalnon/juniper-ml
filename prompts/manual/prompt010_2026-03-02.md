# Perform Full Documentation Audit, all Repos

## Task Overview

### Plan the audit

develop a detailed plan to perform a full, comprehensive audit of all documentation files for all of the included applications.
the plan should be written into a markdown file located in the Juniper/notes/ directory
the plan should include all steps required to perform the following tasks for each application:

- use subagents and memory files to evaluate the current state of all documentation files
- validate the current state of the codebase, configuration, testing, and utility script infrastructure with respect to the documentation files.
- validate all document links including
  - internal links (same document)
  - external documentation links (other documentation files in same application)
  - cross application links (documentation files in other applications)
  - global links (remote documents)
- document all identified problems with links
- document all identified cases of divergence or discrepancies between these two sources.
- document all instances of obsolete information in the existing documentation files.
- ensure all referenced directory structures and filenames are correct
- ensure all procedures are correct and accurately reflect the current state of the codebase

#### Applications included

the following applications should be included:

- juniper-canopy
- juniper-cascor
- juniper-cascor-client
- juniper-cascor-worker
- juniper-data
- juniper-data-client
- juniper-deploy
- juniper-ml

the following applications should be excluded:

- juniper-legacy

#### Directories

In all application directories:

#### Helper scripts

Write helper scripts (e.g., in python or bash) to assist in the data gathering and performance of this audit.

### Prioritize and organize audit tasks

break the doc audit plan tasks into phases.
break each phase up into steps
break each step up into tasks

update the plan with this prioritization and organizational information.
