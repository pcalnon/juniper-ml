# Add dependency documentation step to CI

## Dependency types

add a step to the CI for all repos to update the environment documentation for:

- conda / mamba managed packages and dependencies
- pip managed packages and dependencies
- juniper project packages that are dependencies
- any other dependencies or packages required for development, testing, pre-commit, ci, deployment, or normal operation

## Dependency Documentation Directives

- ensure that the pip managed dependencies are tracked in a pip_requirements_ci.txt file in the conf/ directory
- ensure that the conda/mamba managed dependencies are tracked in a conda_environment_ci.yaml file in the conf/ directory
- ensure that all other dependencies are tracked in a "\<Repo Name>"_OTHER_DEPENDENCIES.md file in the notes/ directory
  - all other dependencies should have their management method included in the OTHER_DEPENDENCIES.md file.
  - these management methods can include, but are not limited to, the following:
    - apt
    - uv
    - npm
- standardize the names of these dependency documentation files across all repos.

## Dependency File Generation

### conda_environment_ci.yaml

Documentation file generation for conda_environment_ci.yaml:

Utilize the following command where "\<Repo Name>" is replaced by the name of the current Repo:

```bash
conda list --explicit >> <Repo Name>/conf/conda_environment_ci.yaml
```

### requirements_ci.yaml

Documentation file generation for requirements_ci.txt:

Utilize the following command where "\<Repo Name>" is replaced by the name of the current Repo:

```bash
pip list --format=freeze > <Repo Name>/conf/requirements_ci.txt
```

## File Specific Requirements

### Conda / Mamba Yaml File

**1. Add header to file:**

- During the Dependency Documentation step of CI, add the following file contents as a header for the conda_environment_ci.yaml file:
  - Juniper/juniper/notes/CONDA_DEPENDENCY_FILE_HEADER.md
- Ensure a copy of this header file is present in all repo's notes/ directories.
  - These header files should be customized for each repo by replacing fields offset by "<" and ">" characters.
    - e.g., "\<Repo Name>" should be replaced by "juniper-cascor" in the Juniper/juniper-cascor/notes/CONDA_DEPENDENCY_FILE_HEADER.md file.
  - Header file customization should be performed for any field offset by "<" and ">" that is constant for each repo
- Fields offset by "<" and ">" that are not constants for repos, should be populated when the new conda_environment_ci.yaml file is generated during CI.
  - These non-constant fields can include, but are not limited to:
    - last modified date
    - copyright, current year
    - python version
    - repo version

**2. Preserve file being replaced:**

- Prior to updating the conda file, add an infix date_time stamp to the existing file's filename between the "conda_environment-ci" and the file extension ".yaml".
  - e.g., conda_environment_ci.yaml --> conda_environment_ci_YYYY-MM-dd_HH:mm:ss.yaml
