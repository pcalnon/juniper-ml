# Pre-Commit ShellCheck Fix Plan

**Date**: 2026-03-13
**Branch**: deployment/new_release/pypi
**Status**: Implementation

---

## Problem Summary

`pre-commit run --all-files` fails on the `shellcheck` hook with 5 warnings across 2 files.
Additionally, 10 unit tests in `tests/test_wake_the_claude.py` fail due to a related
bug in `scripts/wake_the_claude.bash`.

---

## Root Cause Analysis

### ShellCheck Failures (pre-commit hook)

| File | Line(s) | Code | Issue |
|------|---------|------|-------|
| `scripts/activate_conda_env.bash` | 26, 36 | SC1090 | Non-constant source path; shellcheck can't follow |
| `scripts/cleanup_open_worktrees.bash` | 5 | SC2034 | `OLD_IFS` set but never used (IFS never restored) |
| `scripts/cleanup_open_worktrees.bash` | 13 | SC2164 | `cd` without `\|\| exit` fallback |
| `scripts/cleanup_open_worktrees.bash` | 22 | SC2164 | `cd` without `\|\| continue` fallback |

### Test Failures (10 tests in `test_wake_the_claude.py`)

**Root cause**: `generate_uuid()` in `wake_the_claude.bash` stores the UUID in a local
variable and returns an exit code, but never echoes the UUID to stdout. The caller uses
`generated_uuid="$(generate_uuid)"` expecting the UUID on stdout.

Additionally, `debug_log` calls inside `generate_uuid` write to stdout (not stderr),
polluting the captured output when debug mode is enabled.

**Affected tests**:

- `test_default_interactive_script_forwards_expected_defaults`
- `test_session_id_without_value_uses_python_fallback_when_uuidgen_and_proc_fail`
- `test_session_id_without_value_uses_uuid_fallback_when_uuidgen_fails`
- `test_default_launcher_does_not_skip_permissions`
- `test_default_launcher_executes_with_expected_default_arguments`
- `test_default_launcher_forwards_skip_permissions_when_opted_in`
- `test_default_launcher_opt_in_skip_permissions`
- `test_default_launcher_runtime_honors_skip_permissions_opt_in`
- `test_default_launcher_runtime_omits_skip_permissions_by_default`
- `test_session_id_autogenerated_flag_and_value_are_separate`

All 10 tests fail because UUID generation silently produces empty/garbage output,
causing session ID save to fail, which triggers `usage()` and exits with code 1.

---

## Fix Plan

### Fix 1: SC1090 in `activate_conda_env.bash`

Add `# shellcheck source=/dev/null` directive before each `source "${conda_activate_script}"` line (lines 26 and 36).

### Fix 2: SC2034 + SC2164 in `cleanup_open_worktrees.bash`

- **SC2034**: Add `IFS="${OLD_IFS}"` after the `done` (line 37) to restore IFS, making `OLD_IFS` used.
- **SC2164 (line 13)**: Change `cd "${ORIGINAL_PWD}"` to `cd "${ORIGINAL_PWD}" || exit`.
- **SC2164 (line 22)**: Change `cd "${WORK_DIR}"` to `cd "${WORK_DIR}" || continue`.

### Fix 3: `generate_uuid()` in `wake_the_claude.bash`

- Add `echo "${generated_uuid}"` before each `return "${TRUE}"` in all three fallback branches.
- Redirect `debug_log` calls inside `generate_uuid` to stderr: `debug_log "..." >&2`.
